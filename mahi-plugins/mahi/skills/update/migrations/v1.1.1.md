# Migration v1.1.1 — Suppression du répertoire .index/

## Contexte

La migration v1.1.0 créait un répertoire `.mahi/work/.index/<flowId>` pour chaque
workflow. Cette approche a été remplacée : `WorkflowStore` utilise désormais
`registry.json` (déjà présent) comme source de vérité pour résoudre les paths.

Le répertoire `.index/` est donc orphelin et doit être supprimé.

---

## Étape 1 : Supprimer `.mahi/work/.index/`

Vérifier si le répertoire existe :
```bash
ls .mahi/work/.index/ 2>/dev/null && echo EXISTS || echo ABSENT
```

Si présent :
```bash
rm -rf .mahi/work/.index
echo ".mahi/work/.index/ supprimé."
```

Si absent : ignorer.

---

## Étape 2 : Validation

```bash
echo "=== Validation migration v1.1.1 ==="
[ -d .mahi/work/.index ] && echo "ERREUR : .index/ toujours présent" || echo "OK : .index/ absent"
```
