# Migration v1.1.0 — Réorganisation du répertoire .mahi/

## Contexte

Cette migration restructure le répertoire `.mahi/` pour unifier tous les artefacts
sous `.mahi/work/<type>/YYYY/MM/<id>/`.

| Ancien chemin | Nouveau chemin |
|---------------|----------------|
| `.mahi/local/` | `.mahi/.local/` |
| `.mahi/specs/YYYY/MM/<id>/` | `.mahi/work/spec/YYYY/MM/<id>/` |
| `.mahi/decisions/YYYY/MM/<id>/` | `.mahi/work/adr/YYYY/MM/<id>/` |
| `.mahi/artifacts/<id>/` | fusionné avec le répertoire du workflow |
| `.mahi/registry.json` | `.mahi/work/registry.json` |
| `.gitignore` : `.mahi/local/` | `.mahi/.local/` |

---

## Étape 1 : Déplacer `.mahi/local/` → `.mahi/.local/`

Vérifier si `.mahi/local/` existe :
```bash
ls .mahi/local/ 2>/dev/null && echo EXISTS || echo ABSENT
```

Si présent :
```bash
mv .mahi/local .mahi/.local
```

Si absent : ignorer.

## Étape 3 : Fusionner `.mahi/artifacts/` dans les répertoires workflow

Vérifier si `.mahi/artifacts/` existe :
```bash
ls .mahi/artifacts/ 2>/dev/null && echo EXISTS || echo ABSENT
```

Si présent : pour chaque sous-répertoire `<id>` dans `.mahi/artifacts/`, copier son contenu dans le répertoire workflow correspondant :

```bash
for artifactDir in .mahi/artifacts/*/; do
  flowId=$(basename "$artifactDir")
  # Chercher le répertoire workflow correspondant dans .mahi/work/
  targetDir=$(find .mahi/work -maxdepth 4 -type d -name "$flowId" 2>/dev/null | head -1)
  if [ -n "$targetDir" ]; then
    cp -n "$artifactDir"*.md "$targetDir/" 2>/dev/null || true
    echo "Artefacts de $flowId fusionnés dans $targetDir"
  else
    echo "ATTENTION : aucun répertoire workflow trouvé pour $flowId — artefacts laissés en place"
  fi
done
```

Après vérification manuelle, supprimer le répertoire artifacts :
```bash
rm -rf .mahi/artifacts
```

Si absent : ignorer.

## Étape 5 : Créer les fichiers d'index pour WorkflowStore

Le nouveau `WorkflowStore` résout les workflows via `.mahi/work/.index/<flowId>`.
Il faut créer un fichier index pour chaque workflow existant.

```bash
mkdir -p .mahi/work/.index

# Specs
for dir in .mahi/work/spec/*/*/*/; do
  [ -f "$dir/context.json" ] || continue
  flowId=$(basename "$dir")
  # Extraire le subpath relatif à .mahi/work/ (ex: spec/2026/04/my-spec)
  subPath=$(echo "$dir" | sed 's|\.mahi/work/||' | sed 's|/$||')
  echo -n "$subPath" > ".mahi/work/.index/$flowId"
  echo "Index créé : $flowId → $subPath"
done

# ADR
for dir in .mahi/work/adr/*/*/*/; do
  [ -f "$dir/context.json" ] || continue
  flowId=$(basename "$dir")
  subPath=$(echo "$dir" | sed 's|\.mahi/work/||' | sed 's|/$||')
  echo -n "$subPath" > ".mahi/work/.index/$flowId"
  echo "Index créé : $flowId → $subPath"
done
```

## Étape 6 : Ajouter `createdAt` dans chaque `context.json`

Le champ `createdAt` est requis pour les futurs appels `save()` du serveur.
Dériver la valeur depuis `updatedAt` si absent.

Lire chaque `context.json` et vérifier la présence de `createdAt` :

```bash
find .mahi/work -name "context.json" | while read f; do
  hasCreatedAt=$(python3 -c "import json,sys; d=json.load(open('$f')); print('YES' if 'createdAt' in d else 'NO')" 2>/dev/null)
  echo "$f : createdAt=$hasCreatedAt"
done
```

Pour chaque `context.json` sans `createdAt`, l'ajouter (valeur = `updatedAt` si présent, sinon date courante) :

```bash
find .mahi/work -name "context.json" | while read f; do
  python3 - "$f" <<'PYEOF'
import json, sys
from datetime import datetime, timezone

path = sys.argv[1]
with open(path) as fp:
    data = json.load(fp)

if 'createdAt' not in data:
    # Utiliser updatedAt si disponible, sinon maintenant
    created = data.get('updatedAt', datetime.now(timezone.utc).isoformat())
    data['createdAt'] = created
    with open(path, 'w') as fp:
        json.dump(data, fp, indent=2, ensure_ascii=False)
    print(f"  createdAt ajouté dans {path} : {created}")
PYEOF
done
```
